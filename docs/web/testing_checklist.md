### Reduced Motion Support

**Prerequisite:** Enable `prefers-reduced-motion: reduce` in browser/OS settings.

**macOS/iOS:** System Preferences → Accessibility → Display → Reduce motion (ON)

**Windows:** Settings → Ease of Access → Display → Show animations (OFF)

- [ ] Hero canvas animation stops (single static frame displays)
- [ ] Scroll behavior is instant (not smooth)
- [ ] Arrow animation (hero CTA) stops
- [ ] Progress bar still fills on scroll (not animated)
- [ ] Text content remains visible (animations are omitted, not content)

**Expected:** All animations halt; reduced-motion CSS media query is respected (see support.js & index.html styles).

### Performance (Lighthouse / Manual)

- [ ] Page load time: <2 seconds (on 4G, DevTools throttle)
- [ ] Cumulative Layout Shift (CLS) <0.1
- [ ] Largest Contentful Paint (LCP) <2.5s
- [ ] First Input Delay (FID) <100ms

**Test via:**

### Accessibility Checklist

- [ ] All headings have proper hierarchy (h1 > h2 > h3)
- [ ] Images have alt text (or `aria-hidden` if decorative)
- [ ] Links have descriptive text (not just "click here")
- [ ] Color contrast passes WCAG AA: text on background should be ≥4.5:1
  - Light text (#D9D3C7) on dark background (#0A0D14): **✓ high contrast**
  - Teal (#4FD8CE) on dark: **✓ accessible**
- [ ] Buttons are keyboard-accessible (Tab navigation works)
- [ ] Focus outlines are visible (test by pressing Tab repeatedly)
- [ ] Form inputs (if any) have labels
- [ ] Screen reader test (optional): Read page with NVDA/JAWS; narration is logical & complete

**Tools:**
- axe DevTools browser extension (automated a11y checks)
- WebAIM contrast checker (https://webaim.org/resources/contrastchecker/)
