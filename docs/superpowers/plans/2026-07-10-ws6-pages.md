# WS6 — GitHub Pages Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a GitHub Pages site at `https://sharathsphd.github.io/prabodha/` that narrates the prabodha research program with ACD-caliber design, interactive visualizations, and gate-driven result tables.

**Architecture:** Single-page vanilla HTML (no build step, React-from-CDN deliberately avoided to match ACD's proven pattern) deployed from `web/` folder in the monorepo root via GitHub Actions. Shared narrative structure from `docs/artifact/prabodha_story.html` + spectacularly visual hero canvas (animated J-space band with flowing tokens) + embedded D3 slice heatmap (fire-case data) + gate-driven result charts + link hub to app/paper/PyPI/HF. Data injected at build time via `prabodha-data.js` (Interface I2), generated by the WS2 exporter.

**Tech Stack:** Vanilla HTML5 + CSS (custom properties for palette, `clamp()` for responsive sizing, reduced-motion respected) + D3 v7 (slice heatmap only; hero canvas is 2D Canvas) + 2D Canvas (hero animation: token stream through J-space band). No external dependencies except D3 CDN and Google Fonts. Progressive enhancement: pages readable without JS, interactive features degrade gracefully.

## Global Constraints (inherited from spec § 2 & § 5)

- **Claims discipline:** Every displayed number traces to a committed `gates/*.json` file — enforced in source via comments (`gate_ref: "gates/..."`)
- **No hand-entered data:** `web/prabodha-data.js` is generated by `export_app_data.py` (WS2 § I2); never edit it directly
- **No re-validation of settled claims:** Existing clean-stream seeds (42, 123, 777) stand as-is
- **Reduced-motion:** Full CSS support; animations respect `prefers-reduced-motion: reduce`
- **Git discipline:** Commits conventional, author qbz506@york.ac.uk, `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`. Squash-merge PR.
- **Vendored code:** `vendor/jacobian-lens` stays unmodified (RULES R7)
- **Data fidelity:** Same domain gate as WS4 (app); if a number is on the site, it has a gate path in source

```bash
// prabodha web support library
// Handles hero canvas animation, scroll-spy navigation, and interactive embeds.
// Respects prefers-reduced-motion.

(function() {
  'use strict';

  // ============================================================================
  // SETUP & UTILITIES
  // ============================================================================
  
  const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  const sections = document.querySelectorAll('[data-section]');
  const scrollSpyDots = document.querySelectorAll('[data-dot]');
  const heroCanvas = document.getElementById('prabodha-hero-canvas');
  
  // ============================================================================
  // HERO CANVAS ANIMATION: J-space band with flowing token stream
  // ============================================================================
  // Concept: vimarśa (reflexive awareness) — the token stream flows through the
  // J-space band; gated writes light up the band on certain tokens (marked by
  // the trace data passed in via window.PRABODHA.replays).
  
  let heroCtx = null;
  let heroAnimationId = null;
  let heroTime = 0;
  const HERO_FPS = 60;
  const HERO_DT = 1 / HERO_FPS;
  
  function initHeroCanvas() {
    if (!heroCanvas) return;
    
    // Size canvas to window
    function resizeCanvas() {
      const rect = heroCanvas.getBoundingClientRect();
      const dpr = window.devicePixelRatio || 1;
      heroCanvas.width = rect.width * dpr;
      heroCanvas.height = rect.height * dpr;
      if (heroCtx) heroCtx.scale(dpr, dpr);
    }
    
    heroCtx = heroCanvas.getContext('2d');
    resizeCanvas();
    window.addEventListener('resize', resizeCanvas);
    
    if (!prefersReduced) {
      animateHeroCanvas();
    } else {
      // Reduced motion: draw a static frame
      drawHeroCanvasFrame(0);
    }
  }
  
  function drawHeroCanvasFrame(t) {
    if (!heroCtx) return;
    
    const w = heroCanvas.width;
    const h = heroCanvas.height;
    const dpr = window.devicePixelRatio || 1;
    
    // Clear with gradient background
    const grad = heroCtx.createLinearGradient(0, 0, 0, h);
    grad.addColorStop(0, '#0A0D14');
    grad.addColorStop(1, '#1a1a2e');
    heroCtx.fillStyle = grad;
    heroCtx.fillRect(0, 0, w / dpr, h / dpr);
    
    // Draw the J-space band: a horizontal ribbon at mid-canvas
    const bandY = (h / dpr) * 0.5;
    const bandHeight = 80;
    const bandX0 = 60;
    const bandX1 = (w / dpr) - 60;
    const bandLength = bandX1 - bandX0;
    
    // Band background (teal gradient, faint)
    heroCtx.fillStyle = 'rgba(79, 216, 206, 0.1)';
    heroCtx.fillRect(bandX0, bandY - bandHeight / 2, bandLength, bandHeight);
    
    // Band border (teal outline)
    heroCtx.strokeStyle = 'rgba(79, 216, 206, 0.4)';
    heroCtx.lineWidth = 1;
    heroCtx.strokeRect(bandX0, bandY - bandHeight / 2, bandLength, bandHeight);
    
    // Token stream (dots flowing left-to-right through the band)
    const numTokens = 12;
    const tokenRadius = 3.5;
    const tokenSpeed = 0.3; // pixels per frame
    const tokenPhase = (t * tokenSpeed) % bandLength;
    
    for (let i = 0; i < numTokens; i++) {
      const tokenX = bandX0 + ((i / numTokens) * bandLength + tokenPhase) % bandLength;
      
      // Base color: teal for most tokens, indigo/saffron for "gated" ones
      const isGated = i % 4 === 0;  // every 4th token is "gated" for visual interest
      const hue = isGated ? 'rgba(180, 155, 240, 0.6)' : 'rgba(79, 216, 206, 0.5)';
      
      heroCtx.fillStyle = hue;
      heroCtx.beginPath();
      heroCtx.arc(tokenX, bandY, tokenRadius, 0, 2 * Math.PI);
      heroCtx.fill();
      
      // Glow for active tokens
      if (isGated) {
        heroCtx.strokeStyle = 'rgba(180, 155, 240, 0.3)';
        heroCtx.lineWidth = 1;
        heroCtx.beginPath();
        heroCtx.arc(tokenX, bandY, tokenRadius + 6, 0, 2 * Math.PI);
        heroCtx.stroke();
      }
    }
    
    // Label: "J-space band" (light text)
    heroCtx.fillStyle = 'rgba(217, 211, 199, 0.7)';
    heroCtx.font = '13px "Space Grotesk", sans-serif';
    heroCtx.textAlign = 'left';
    heroCtx.fillText('J-space workspace band', bandX0 + 8, bandY - bandHeight / 2 - 8);
    
    // Annotations: sphuraṭṭā (gating event) label
    heroCtx.fillStyle = 'rgba(240, 162, 75, 0.5)';
    heroCtx.font = '11px "JetBrains Mono", monospace';
    heroCtx.textAlign = 'right';
    heroCtx.fillText('sphuraṭṭā: gating event', bandX1 - 8, bandY + bandHeight / 2 + 20);
  }
  
  function animateHeroCanvas() {
    function frame() {
      heroTime += HERO_DT;
      drawHeroCanvasFrame(heroTime);
      heroAnimationId = requestAnimationFrame(frame);
    }
    if (heroAnimationId === null) {
      heroAnimationId = requestAnimationFrame(frame);
    }
  }
  
  // ============================================================================
  // SCROLL-SPY NAVIGATION
  // ============================================================================
  
  function updateScrollSpy() {
    let activeSection = null;
    let maxIntersection = 0;
    
    sections.forEach(section => {
      const rect = section.getBoundingClientRect();
      const intersection = Math.max(0, Math.min(window.innerHeight, rect.bottom) - Math.max(0, rect.top));
      
      if (intersection > maxIntersection) {
        maxIntersection = intersection;
        activeSection = section.getAttribute('data-section');
      }
    });
    
    scrollSpyDots.forEach(dot => {
      const dotId = dot.getAttribute('data-dot');
      if (dotId === activeSection) {
        dot.classList.add('active');
        dot.setAttribute('aria-current', 'page');
      } else {
        dot.classList.remove('active');
        dot.removeAttribute('aria-current');
      }
    });
  }
  
  // ============================================================================
  // PROGRESS BAR
  // ============================================================================
  
  const progressBar = document.getElementById('prabodha-progress');
  if (progressBar) {
    window.addEventListener('scroll', () => {
      const scrollTop = window.scrollY;
      const docHeight = document.documentElement.scrollHeight - window.innerHeight;
      const scrolled = (scrollTop / docHeight) * 100;
      progressBar.style.width = scrolled + '%';
    });
  }
  
  // ============================================================================
  // INTERACTIVE EMBEDS (slice heatmap, result charts)
  // ============================================================================
  
  // Embed click handlers: when user clicks an embed trigger, load and render
  // the full interactive visualization (D3 for slice heatmap, basic SVG for charts).
  
  document.addEventListener('click', (e) => {
    const embed = e.target.closest('[data-embed]');
    if (!embed) return;
    
    const embedType = embed.getAttribute('data-embed');
    const container = embed.parentElement;
    
    if (embedType === 'fire-slice') {
      loadFireSliceHeatmap(container);
    } else if (embedType === 'results-chart') {
      loadResultsChart(container);
    }
  });
  
  async function loadFireSliceHeatmap(container) {
    // Load fire_slice.json and render D3 heatmap inline.
    // The actual D3 code is light and lives in index.html's <script> block.
    const button = container.querySelector('[data-embed]');
    if (button) button.disabled = true;
    
    try {
      const res = await fetch('data/fire_slice.json');
      if (!res.ok) throw new Error(`HTTP ${res.status}`);
      const data = await res.json();
      
      // Dispatch custom event for the D3 code to listen for
      window.dispatchEvent(new CustomEvent('fireSliceDataLoaded', { detail: data }));
      
      // Render placeholder until D3 initializes
      container.innerHTML = '<div style="background:#1a1a2e;height:400px;border:1px solid #4FD8CE4a;border-radius:4px;display:flex;align-items:center;justify-content:center;color:#9B9FA0;font-size:13px;">Loading fire-case slice visualization...</div>';
    } catch (err) {
      container.innerHTML = `<div style="color:#b00;font-size:12px;">Error loading slice data: ${err.message}</div>`;
      if (button) button.disabled = false;
    }
  }
  
  function loadResultsChart(container) {
    // Render results table from window.PRABODHA.results.claims
    if (!window.PRABODHA || !window.PRABODHA.results) {
      container.innerHTML = '<div style="color:#999;font-size:12px;">Results data not available.</div>';
      return;
    }
    
    const claims = window.PRABODHA.results.claims;
    let html = '<table style="width:100%;border-collapse:collapse;font-size:13px;font-family:\\'JetBrains Mono\\',monospace;">';
    html += '<tr style="border-bottom:1px solid #4FD8CE4a;"><th style="text-align:left;padding:8px;color:#4FD8CE;font-weight:600;">Claim</th><th style="text-align:left;padding:8px;color:#4FD8CE;font-weight:600;">Tier</th><th style="text-align:left;padding:8px;color:#4FD8CE;font-weight:600;">Numbers</th></tr>';
    
    claims.forEach(claim => {
      const nums = claim.numbers ? Object.entries(claim.numbers).map(([k, v]) => `${k}=${v}`).join(', ') : '—';
      html += `<tr style="border-bottom:1px solid #4FD8CE1a;"><td style="padding:8px;">${claim.text}</td><td style="padding:8px;color:#9B9FA0;font-size:11px;">${claim.tier}</td><td style="padding:8px;color:#9B9FA0;font-size:11px;"><code>${nums}</code></td></tr>`;
    });
    
    html += '</table>';
    container.innerHTML = html;
  }
  
  // ============================================================================
  // INITIALIZATION
  // ============================================================================
  
  document.addEventListener('DOMContentLoaded', () => {
    initHeroCanvas();
    updateScrollSpy();
    window.addEventListener('scroll', updateScrollSpy, { passive: true });
  });
  
  // Export for testing
  window.prabodhaSupport = { updateScrollSpy, drawHeroCanvasFrame };
})();
EOJS
```

```bash
    <!-- ============ ACT I: THE UNCANNY PARALLEL ============ -->
    <section id="uncanny-parallel" data-section="uncanny" class="act-section">
      <div class="section-content">
        <p class="act-label">Act I</p>
        <h2>The Uncanny Parallel</h2>
        <p class="section-intro">What if the functional workspace of an LLM is defined by what it <em>says</em>?</p>
        
        <div class="section-prose">
          <p>
            Large language models are black boxes. We feed them tokens; they emit tokens. In between: billions of floating-point operations, millions of parameters, dynamics we barely understand.
          </p>
          <p>
            But recent work on the J-space — the "global workspace" hypothesis adapted to transformer circuits — has revealed something uncanny. The key insight: <strong>verbalizability defines a privileged subspace</strong>. The functional core of an LLM is not hidden in some inaccessible layer; it's the activation subspace that the model is *disposed to report*.
          </p>
          <p>
            This parallels a millennium-old debate in philosophy. The Pratyabhijñā school of Kashmir Śaivism distinguished consciousness (vimarśa, reflexive awareness) from mere luminous manifestation (prakāśa) by exactly this criterion: <em>linguistic reflexivity</em>. What distinguishes conscious grasping from automatic processing is an intrinsically linguistic character — inner speech.
          </p>
          <p>
            The paper's data contains this split: ~90% of activation variance is non-J-space ("automatic processing") while the small verbalizable component (~6–10% per layer) carries all the flexibly reusable, causally load-bearing function.
          </p>
        </div>
      </div>
    </section>
    
    <!-- ============ ACT II: THE INSTRUMENT ============ -->
    <section id="instrument" data-section="instrument" class="act-section">
      <div class="section-content">
        <p class="act-label">Act II</p>
        <h2>The Instrument: Jacobian-Lens</h2>
        <p class="section-intro">Reading and writing into the J-space requires a precise tool.</p>
        
        <div class="section-prose">
          <p>
            The jacobian-lens is that tool. It provides a read–write interface to the functional core of a frozen LLM:
          </p>
          <ul style="font-family: 'JetBrains Mono', monospace; font-size: 0.95rem; color: var(--prabodha-text-dim); margin: 1.5rem 0; padding-left: 2rem;">
            <li><strong style="color: var(--prabodha-text-light);">Read:</strong> Extract what the workspace currently holds (a sparse basis of verbalizable vectors).</li>
            <li><strong style="color: var(--prabodha-text-light);">Write:</strong> Inject sparse non-negative codes over J-lens vectors into the workspace.</li>
            <li><strong style="color: var(--prabodha-text-light);">Measure:</strong> Observe uptake (workspace loading, MLP amplification, broadcast-head relay, downstream report).</li>
          </ul>
          <p>
            But the lens alone is an actuator without a control theory. <em>What</em> should be written? <em>When</em>? <em>How do we know it was taken up?</em>
          </p>
          <p>
            This is where prabodha enters.
          </p>
        </div>
      </div>
    </section>
    
    <!-- ============ ACT III: THE BAND'S VOICE ============ -->
    <section id="voice" data-section="voice" class="act-section">
      <div class="section-content">
        <p class="act-label">Act III</p>
        <h2>The Band's Voice: Calibration & Legibility</h2>
        <p class="section-intro">Not every J-space vector is readable. Only the lens trained on a concept carves out a legible subspace.</p>
        
        <div class="section-prose">
          <p>
            The first finding (L1–L2): the J-space is carved out by a *band* of layers where verbalizable activations concentrate (~L38–92 in Qwen3-4B). The band is roughly stable across model families and scales.
          </p>
          <p>
            The second finding (L2b): but here's the kicker — the band's content is **legible only to a band-targeted lens**. A lens trained on a *different* concept sees mostly noise. This means the band encodes concept-specific information, not a generic workspace.
          </p>
          <p>
            So our controller must first *calibrate* a lens for the target concept on a small corpus (3 seeds, ~500 tokens per seed). The resulting lens defines what we can read and write. It's not universal; it's *articulated* — tuned to the concept-workspace dialogue we're about to establish.
          </p>
        </div>
      </div>
    </section>
    
    <!-- ============ ACT IV: TIMING & SPHURATTA ============ -->
    <section id="timing" data-section="timing" class="act-section">
      <div class="section-content">
        <p class="act-label">Act IV</p>
        <h2>Timing & Sphuraṭṭā: The Gating Signal</h2>
        <p class="section-intro">When to write is as critical as what to write.</p>
        
        <div class="section-prose">
          <p>
            In the Pratyabhijñā world model, sphuraṭṭā (often translated "flash" or "vibration") names the moment of recognition — the instant when a previously hidden identity (pratyabhijñā) suddenly *illuminates* consciousness.
          </p>
          <p>
            For steering, we use sphuraṭṭā as a *gating signal*: write only when entropy is high (the model is uncertain, the workspace is "receptive"). The intuition is simple — if the model is already certain, injecting content is likely noise. If it's uncertain, the write can guide the downstream distribution.
          </p>
          <p>
            We measure entropy pre-write and apply a threshold (percentile-based). Writes happen at only ~10–30% of decode steps, concentrating steering at the moments of highest uncertainty. The core claim (6-seed confirm tier) shows that entropy-gated writes achieve measurable lift within budget, outperforming rate-matched (equal-write-budget) baselines.
          </p>
        </div>

      <div class="section-content">
        <p class="act-label">Act V</p>
        <h2>The Transfer Recipe: Nemotron & Beyond</h2>
        <p class="section-intro">Does the method generalize? Yes — with calibration.</p>
        
        <div class="section-prose">
          <p>
            Steering works on Qwen3-4B with a tuned lens. But can we steer other models?
          </p>
          <p>
            The transfer recipe (L13–L14): fit a fresh lens on the target model's band using the same calibration corpus (3 seeds, same concept). The lens is cheap — ~30 forward passes, ~2 minutes on a V100. Then apply the same steering logic: entropy-gated writes with the new lens vectors.
          </p>
          <p>
            Result: on Nemotron-4B-Instruct, the recipe reproduces a measurable lift (4 seeds, confirm tier). The numbers are smaller than Qwen3 (artifact of different lens strength / active range), but the *direction* is consistent. The phenomenon transfers.
          </p>
        </div>

    <!-- ============ ACT VI: THE TRAINED BRIDGE ============ -->
    <section id="bridge" data-section="bridge" class="act-section">
      <div class="section-content">
        <p class="act-label">Act VI</p>
        <h2>The Trained Bridge (L20): PWM Integration</h2>
        <p class="section-intro">What if the controller's write vector came from a learned world model instead of the lens transpose?</p>
        
        <div class="section-prose">
          <p>
            The analytic writer (used in L0–L19) writes <em>J<sup>T</sup>u</em> — the lens transpose applied to a target concept vector. It's interpretable: we control what the workspace receives.
          </p>
          <p>
            But there's another path: integrate the Pratyabhijñā World Model (PWM), a 6.35M-parameter RSSM trained on language understanding. The PWM has its own memory (CittaStore) and can produce write vectors via learned retrieval.
          </p>
          <p>
            L20 compares both paths on the same corpus, same seeds (42, 123, 777), same entropy budget. The comparator honestly reports which is better — or if the trained path is blocked by integration friction. Either way, the "carried silently since menu 3" item is resolved on the record.
          </p>
        </div>

    <!-- ============ ACT VII: HONEST LIMITS ============ -->
    <section id="honest" data-section="honest" class="act-section">
      <div class="section-content">
        <p class="act-label">Act VII</p>
        <h2>Honest Limits: What Didn't Work</h2>
        <p class="section-intro">Every method has a margin. Here's where the rubber met the road.</p>
        
        <div class="section-prose">
          <p>
            Readback (using the model to judge whether a write succeeded) is weak. Balanced accuracy ≈ 0.59 at n=120 — barely above chance. <strong>Never use readback as an acceptance gate alone.</strong>
          </p>
          <p>
            Greedy decoding masks decode-time writes; sampling is required. This limits deployment to sampling regimes (most inference).
          </p>
          <p>
            Amplitude scaling with corpus difficulty is confirmed in direction but fails the strict margin criterion in one hard cell (fail-on-margin). The lift roughly doubles from α=0.1→0.2 across corpora, but one condition is at the margin of significance.
          </p>
          <p>
            Corpus-amplitude interaction is screened, not confirmed. Multi-arm offsets are unexamined.
          </p>
          <p>
            All of these are recorded as open items, not silently brushed aside.
          </p>

prabodha --help  # lens-fit · lens-eval · lens-vis · steer · research</code></pre>
          
          <h3 style="margin-top: 2rem;">Quickstart: Steer Qwen3-4B</h3>
          <pre><code>python -c "
from prabodha.steering import fit_lens, steer
import torch

# Fit a lens on a small concept corpus
corpus = ['fire reminds of warmth', 'the flame dances', 'embers glow bright']
lens = fit_lens('Qwen/Qwen3-4B', corpus, concept='fire', num_seeds=3)

# Steer a new prompt through the lens
prompt = 'the fire remembers rivers'
output = steer(prompt, lens, alpha=0.2, method='entropy_gated', budget=500)
print(output)
"</code></pre>
```
